# 概要
　ブロックによるプログラミングをよりフローチャート的に実現するためのプロジェクト。
 　これはプロジェクトで何が良いのかを実験して、良いと思ったものを保存しておくためのリポジトリ。
  
# メモ
・vueflowの公式にある例を参考にプログラムを組んだはずだが、実際に書いたモジュールのインポート部分だけが微妙に違う。どこを参考にしたのか覚えていないため、このリポジトリにすでに書いてあるコードを参考にすること。
・使わない変数が存在すればvueのコードチェッカーによってエラーが投げられる。package.jsonのruleに追記し、エラーを投げないようにしたものの、実際にはチェッカーに引っかからない方が良い。コードがきれいに書けるわけだからね。そんな方法を模索する必要がある。
